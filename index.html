vue中权限控制的实践经验总结

发表于 2022-06-19

前言

作为程序员，最痛苦的不是你在网上搜不到答案，而是搜到了一篇文章后，发现全网都是这篇文章。全是抄袭，文章篇幅非常短，没有自己的一个实践和总结。最让人疑惑的是你在搜到一篇百赞以上，好评非常多，然后遇到的问题恰好是你要找的，但是cv到你的代码里就不管用，或者有bug，然后在评论里也找不到任何人提出问题

权限key管理

权限key就是某个权限对应的唯一标识。

权限在数据库中一般有这些字段：

  • id
  • 权限name
  • 权限desc
  • 权限key
  • 权限类型（menu、button、api）
  • 父级权限id

所以说前端页面里需要有权限key管理的增删改查的页面，一般用tree组件去挂载这类数据。

角色管理

有了权限，当然需要角色去关联权限，比如：

  • 超级管理员
    • 增删改查所有权限
  • 测试人员
    • 查增改的权限
  • 普通用户
    • 查的权限

角色管理包括：角色的增删改查，角色与权限关联的增、删操作。

用户管理

有了角色，当然就需要用户去承载角色才有意义。每个用户可以被分配多个角色，当然分配角色的目的就在于在用户登录后返回给前端该用户所拥有的所有权限key

路由（菜单）权限

1、动态路由与静态路由

静态路由就是在new VueRouter()时传入的路由对象

动态路由就是后期通过router.addRoute()进行添加的路由对象

2、为什么需要动态路由

高级路由：需要进行权限校验的路由对象

因为用户的权限key是动态从后端获取的，获取了就需要进行权限校验来判断用户拥有哪些高级路由的权限，并进行addRoute

3、路由权限控制流程图

4、完整路由权限控制代码

按钮权限

按钮权限比较简单，一般是通过全局注册一个指令，然后在指令的inserted方法中，拿到vuex中保存的当前用户所拥有的所有按钮权限key，去判断该用户是否拥有这个按钮上绑定的key值。如果没有就移除掉该button，parentNode.removeChild(button)

指令使用例子：

v-permission='button:role:link'
看看该用户拥有的按钮权限key中是否有button:role:link即可

api权限

api权限一般是在后端进行拦截处理，拿到将当前用户和当前请求的接口进行判断，如果用户拥有当前接口的权限key，那么就放行，如果没有就response 403

webpack中commonjs和esmodule的源码级别分析

发表于 2022-05-23 更新于 2022-06-05

背景

注意：这篇文章只是针对webapck中对模块化的处理，并不涉及到node中的模块化

最近准备面试的八股文，看到了很多关于esmodule和commonjs模块化比较的文章，但是和自己学习webpack过程中关于模块化的原理有所差异。于是写下这篇文章来把那些容易被初学者误解的定义重新梳理一遍。

【误解】关于es模块是在编译时加载（静态加载），commonjs模块是在运行时加载

其实在webpack中不管是commonjs模块还是es模块都是在编译时加载到内存的，
只不过es模块的引用会被提升到当前文件的顶部并放在内存中

所以理论上es模块和commonjs模块都是静态加载，所以都可以实现tree-shaking，但是在webpack5之前只对es模块进行导出模块的使用情况的分析。
webpack5已经支持对commonjs模块的tree-shaking：https://webpack.docschina.org/blog/2020-10-10-webpack-5-release/#commonjs-tree-shaking

【误解】关于commonjs模块是同步导入，es模块是异步导入的问题

在webpack中commonjs的确是同步导入的（运行时才知道导入的是什么模块），esmodule默认情况下不是异步导入，只是在编译阶段会把所有import xxx from "module.js"这种导入语句提升到代码开头。

在esmoudle中只有import()函数才是异步导入的，编译结果如下：

./esmodule/index5:

webpack编译后：

【误解】esmodule：遇到模块加载命令import时不会去执行模块其余代码

在webpack中：
import {a,b} from "index.js"时，不仅会生成对导出模块a和模块b的引用，也会执行index.js本身的代码

为什么esmodule导入的变量不能重新赋值修改

commonjs导入的都是通过const {}解构得到的变量，不支持修改

如果强行修改esmodule导出的模块将会报错：
Uncaught TypeError: Cannot set property xxx of #<Object> which has only a getter

背景

因为最近看到很多canvas的骚操作，有画龙的、有做动画的，在看他们代码的时候，这些图形处理的方式让我突然想到在大二用python做的一个隐形斗篷的骚操作，于是二话不说就打开github创建了项目，一顿骚操作下，效果终于出来了！！！

实现过程

1、先看html里面有什么

video用来播放原视频

canvas用来快速绘画（视觉效果约等于播放）原视频画面

2、再看js中的核心代码

CanvasPerspective类

  • videoEle （保存video对象）
  • canvasEle（保存canvas对象）
  • colorScope（目标区域的颜色，默认是红色）
  • context（canvas的绘画话柄）
  • playing（video的播放状态）
  • gap（扫描像素点的step）
  • backgroundImageData（原视频第一帧的背景图数据）

drawVideo方法

  1. 将video当前画面绘画到canvas上

  2. 获取canvas图像数据

  3. 保存第一帧（背景图）的图像数据

    • imageData[ 3 ] !== 0 是在使用iphone safari时发现了一个bug，backgroundImageData的数据为[0, 0, 0, 0, …..],即一个黑色全透明的背景。
    • 所以这里判断一下，当第一个像素点的透明度不为0时才认为是原视频的背景（关于imageData数据的解释和运用在后面）

  4. 扫描canvas的像素点，拿到扫描到像素点的颜色值（[red, green, blue, alpha]）,判断当前像素点是否是目标颜色，如果是目标颜色就拿之前保存的背景图中这个像素点的颜色值去覆盖掉自己。

  5. 利用window.requestAnimationFrame来执行动画

不足

1.由于需要扫描整个图像的数据，一般都是几万个数据的数组，所以性能还有待提升。

2.对于红绿蓝颜色区域的判断不是非常准确，因为实际拍摄的画面有很多影响因素，导致并不能达到标准rgba值。

3.对视频内容有严格要求（当然这是这种设计方案必然会产生的问题）

  1. 必须要保持背景不动
  2. 在视频前面几帧，只能展示背景，不能有其他干扰项

1、attributes

  • i：不区分大小写
  • g：执行全局匹配（当匹配到一个子字符串时，继续匹配下去，直到字符串结尾）
  • m：执行多行匹配（当字符串中存在\n换行符时，多行匹配的意义就出来了）

这里对m进行例子解释，另外两个比较简单

2、pattern

a、元字符

在pattern中除了普通的字符以外还有元字符（也叫做特殊字符）。

  • . 查找单个字符

  • \w 查找单词符号 包括：a-z，A-Z，0-9，以及下划线

  • \W 查找非单词符号

  • \d 查找数字

  • \D 查找非数字字符

  • \s 查找空白字符

    • 空格符 (space character)
    • 制表符 (tab character)
    • 回车符 (carriage return character)
    • 换行符 (new line character)
    • 垂直换行符 (vertical tab character)
    • 换页符 (form feed character)

  • \S 查找非空白字符

  • \b 元字符匹配单词边界

    • 单词就是以空格隔开的字符串

  • \B 匹配非单词边界

  • \n 查找换行符

  • \r 查找回车符

  • \t 查找制表符

b、量词

  • n+ 匹配任何包括至少1个n的字符串

  • n* 匹配任何包括0个或多个n的字符串

  • n? 匹配任何包括0个或者1个的字符串

  • n{X} 匹配包含X个n的序列的字符串

  • n{X,Y} 匹配包含X-Y个n的序列的字符串

  • n{X,} 匹配包含至少X个n的序列的字符串

  • n$ 匹配任何结尾是n的序列的字符串

  • ^n 匹配任何开头是n的序列的字符串

  • ?=n 匹配任何其后紧跟n的序列的字符串

  • ?<n 匹配任何其前紧跟n的序列的字符串

    • 注意：没有?>n
  • ?!n 匹配任何其后不紧跟n的序列的字符串

3、描述字符范围

  • [abc]：查找方括号内任意一个字符。
  • [^abc]：查找不在方括号内的字符。
  • [0-9]：查找从 0 至 9 范围内的数字，即查找数字。
  • [a-z]：查找从小写 a 到小写 z 范围内的字符，即查找小写字母。
  • [A-Z]：查找从大写 A 到大写 Z 范围内的字符，即查找大写字母。
  • [A-z]：查找从大写 A 到小写 z 范围内的字符，即所有大小写的字母。
